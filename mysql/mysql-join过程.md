### mysql join 过程

mysql 的 join 是拿到两张表的数据 M * N 去比较的



它优化的地方

加入 join_buffer



每次 M 中 拿 部分数据放到 join_buffer 里面

然后用 join_buffer  里的 M 去  N 里比较



然后  join_buffer 去比较 N , N只需要读一次就可以比较完一个 join_buffer 

不需要 M 中一行，就去读一遍 N 



![](img\20220914142941251.png)



尽管 join_buffer 优化但还很耗性能，这也是为什么不建议使用 join 的原因

最好的方式，join 的字段使用索引，那么他们就不需要比较，直接索引定位

建议拆分多次单表操作



### 为什么需要小表驱动大表



mysql 默认是会优化的



在JOIN查询中经常用到的 inner join、left join、right join

问题解答：

1. 当使用left join时，左表是驱动表，右表是被驱动表 ;

2. 当使用right join时，右表时驱动表，左表是被驱动表 ;

3. 当使用inner join时，mysql会选择数据量比较小的表作为驱动表，大表作为被驱动表 ;



为什么？

小表驱动大表 > A驱动表，B被驱动

```
 for(200条){
   for(20万条){
     ...
   }
 }

```

大表驱动小表 > B驱动表，A被驱动表

```
for(20万){
   for(200条){
    ...
   }
 }
```

**总结：**

- 如果小的循环在外层，对于表连接来说就只连接200次 ;
- 如果大的循环在外层，则需要进行20万次表连接，从而浪费资源，增加消耗 ;
